# Vue-Router 路由配置及动态路由

Vue-Router 提供了灵活的路由配置方式，包括静态路由、动态路由、嵌套路由等。不同的配置适用于不同场景。下面详细说明。

## 1. 静态路由

- 定义固定的路径和组件。
- 路径和组件一一对应。

```js
const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About },
];
```

**适用场景**：大多数固定页面，如首页、关于页、登录页。

## 2. 动态路由

- 路径包含参数，用 `:` 声明。
- 可以匹配多个动态路径。

```js
const routes = [{ path: '/user/:id', component: UserProfile }];
```

### 获取参数

```vue
<script setup>
import { useRoute } from 'vue-router';
const route = useRoute();
console.log(route.params.id);
</script>
```

**适用场景**：用户详情页、商品详情页、文章详情页等需要根据 ID 或 slug 渲染不同内容的页面。

## 3. 嵌套路由

- 子路由配置在父路由的 `children` 数组中。
- 可实现页面布局复用。

```js
const routes = [
  {
    path: '/dashboard',
    component: DashboardLayout,
    children: [
      { path: '', component: DashboardHome },
      { path: 'settings', component: DashboardSettings },
    ],
  },
];
```

**适用场景**：后台管理系统、嵌套布局页面。

## 4. 命名路由

- 给路由配置 `name` 属性，方便编程式跳转。

```js
const routes = [{ path: '/about', name: 'About', component: About }];
```

```js
router.push({ name: 'About' });
```

**适用场景**：程序中频繁跳转或动态生成导航的页面。

## 5. 重定向与别名

### 5.1 重定向

```js
{ path: '/home', redirect: '/' }
```

### 5.2 别名

```js
{ path: '/', component: Home, alias: '/index' }
```

**适用场景**：保持旧 URL 的兼容性、路由优化。

## 6. 路由元信息（Meta）

- 配置路由元信息，通过 `to.meta` 获取。

```js
const routes = [
  { path: '/about', component: About, meta: { requiresAuth: true } },
];
```

**适用场景**：权限控制、页面标题设置、导航守卫条件。

## 7. 总结

| 路由类型    | 描述                 | 适用场景                       |
| ----------- | -------------------- | ------------------------------ |
| 静态路由    | 固定路径和组件       | 首页、登录页、关于页等常规页面 |
| 动态路由    | 带参数的路径         | 用户详情页、商品详情页、文章页 |
| 嵌套路由    | 父子关系路由         | 后台管理系统、布局复用页面     |
| 命名路由    | 给路由命名，方便跳转 | 程序频繁跳转、动态生成导航     |
| 重定向/别名 | 旧路径兼容或优化     | URL 重构、旧版本兼容           |
| Meta 信息   | 配置附加信息         | 权限校验、动态标题、导航守卫   |

**使用建议**：

- 普通页面使用静态路由。
- 根据 ID 或动态内容使用动态路由。
- 布局复用使用嵌套路由。
- 权限控制或导航守卫结合 meta 信息。
- 兼容旧 URL 使用重定向或别名。
