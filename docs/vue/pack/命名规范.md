# Vue 组件的命名规范

## 一、命名方式汇总

| 命名类型                 | 推荐格式                                  | 示例                     | 说明                                              |
| ------------------------ | ----------------------------------------- | ------------------------ | ------------------------------------------------- |
| 单文件组件（.vue 文件）  | 大驼峰（PascalCase）                      | `UserCard.vue`           | 符合 Vue 官方推荐，与组件注册保持一致             |
| 全局组件注册名           | 大驼峰或中横线（PascalCase / kebab-case） | `UserCard` / `user-card` | Vue3 推荐 PascalCase，模板中可自动识别 kebab-case |
| 局部组件（在模板中使用） | 中横线（kebab-case）                      | `<user-card />`          | 模板中 HTML 不区分大小写，建议统一使用 kebab-case |
| 目录命名                 | 中横线（kebab-case）                      | `user-profile/`          | 避免空格或大写，提高跨系统兼容性                  |

## 二、命名规则细则

1. **组件名必须多词组合**，防止与原生 HTML 标签冲突。例如：`Button` 不推荐，应使用 `BaseButton`。
2. **基础组件**（通用、无业务逻辑）应以 `Base` 或 `App` 开头，如：`BaseInput`、`AppHeader`。
3. **单页面组件**（路由页面）建议以 `Page` 结尾，如：`LoginPage.vue`。
4. **业务模块组件** 可使用功能前缀，如：`UserCard`、`ProductList`。
5. **组件文件名与组件注册名一致**，避免维护混乱。
6. **自定义指令或插件组件**，使用统一前缀，如公司缩写或项目代号（例：`ac-`）。

## 三、示例

```bash
src/
├── components/
│   ├── BaseButton.vue      # 基础组件
│   ├── UserCard.vue        # 功能组件
│   └── NavBar.vue          # 全局组件
├── views/
│   ├── HomePage.vue        # 页面组件
│   └── LoginPage.vue
└── App.vue
```

模板中使用：

```vue
<template>
  <div>
    <nav-bar />
    <user-card />
    <base-button label="提交" />
  </div>
</template>
```

## 四、总结

Vue 官方推荐遵循统一的命名风格：

- 文件名使用 PascalCase
- 模板标签使用 kebab-case
- 避免 HTML 标签名冲突
- 基础、业务、页面组件分层命名，有助于可维护性和可读性。
