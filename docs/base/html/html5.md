# HTML5 对比 HTML 的更新

**定义：**
HTML5 是 HTML 的最新标准，旨在改进 Web 的结构、表现和交互功能，提供更强的多媒体、离线和语义支持。

## 📘 主要更新方向

HTML5 相比传统 HTML 的改进可以分为以下五个主要方向（点击快速跳转）：

1. [结构化标签增强](#1.-结构化标签增强)
2. [多媒体支持](#2.-多媒体支持)
3. [表单增强](#3.-表单增强)
4. [新增 API](#4.-新增api)
5. [语法变化](#5.-语法变化)

## 1. 结构化标签增强

HTML5 新增了大量语义化结构标签，使页面结构更清晰、可读性更强。[查看常见语义化标签 →](./语义化.md)

| 新标签      | 作用                 |
| ----------- | -------------------- |
| `<header>`  | 定义页面或区块的头部 |
| `<nav>`     | 定义导航区域         |
| `<article>` | 定义独立的内容块     |
| `<section>` | 定义文档中的节或段落 |
| `<aside>`   | 定义附属信息         |
| `<footer>`  | 定义页面或区块的底部 |

**优势：**

- 提升 SEO 效果。
- 提高代码可维护性。
- 更易被屏幕阅读器识别。

## 2. 多媒体支持

HTML5 内置多媒体支持，无需 Flash 等插件。

| 标签      | 功能               |
| --------- | ------------------ |
| `<audio>` | 原生音频播放       |
| `<video>` | 原生视频播放       |
| `<track>` | 视频字幕、文字说明 |

**示例：**

```html
<video controls width="400">
  <source src="demo.mp4" type="video/mp4" />
  您的浏览器不支持 video 标签。
</video>
```

**优势：**

- 原生播放支持，跨平台。
- 更好地与 JS 控制结合。
- 支持多种格式与字幕轨道。

## 3. 表单增强

HTML5 增强了表单功能，新增输入类型与验证属性，减少 JS 校验依赖。

| 新输入类型              | 示例                    |
| ----------------------- | ----------------------- |
| `email`                 | `<input type="email">`  |
| `url`                   | `<input type="url">`    |
| `number`                | `<input type="number">` |
| `range`                 | `<input type="range">`  |
| `date`, `time`, `month` | 日期时间输入            |
| `color`                 | 颜色选择器              |
| `search`                | 搜索输入框              |

**新增属性：** `required`, `placeholder`, `autofocus`, `pattern`, `min`, `max`, `step`。

**优势：**

- 浏览器原生校验。
- 更好的用户交互体验。
- 减少表单验证脚本代码量。

## 4. 新增 API

HTML5 引入了多种新 API，增强 Web 应用的交互性与性能。

| API               | 功能说明                                  |
| ----------------- | ----------------------------------------- |
| Canvas API        | 图形绘制（如游戏、图表）                  |
| Geolocation API   | 获取地理位置                              |
| Web Storage       | 本地存储（localStorage / sessionStorage） |
| WebSocket         | 全双工通信                                |
| Drag and Drop API | 实现拖拽交互                              |
| History API       | 控制浏览历史记录                          |
| Web Workers       | 多线程执行，提高性能                      |

**优势：**

- 支持离线与实时交互。
- 降低后端压力。
- 提升前端性能表现。

## 5. 语法变化

HTML5 对语法进行了简化与规范化。

**语法改进要点：**

1. 简化文档声明：`<!DOCTYPE html>`。
2. 可省略 `<html>`、`<body>` 标签（不推荐）。
3. 新增全局属性：`contenteditable`、`hidden`、`draggable`。
4. 废弃旧标签：`<font>`、`<center>`、`<big>`、`<acronym>` 等。

**优势：**

- 语法更简洁一致。
- 兼容性更高。
- 可读性更强。

## ✅ 总结（面试高频一句话）

> HTML5 在结构、表现和交互层面全面增强，通过引入语义化标签、原生多媒体、表单增强和新 API，使 Web 开发更高效、更强大、更现代化。
