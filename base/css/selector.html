CSS 选择器类型 ​

1. 基本选择器 ​

选择器示例用法说明优缺点
标签选择器div选中所有 <div> 元素✅ 简单、语义清晰；❌ 灵活性低，容易影响全局
类选择器.btn选中 class 为 btn 的元素✅ 灵活、可复用；❌ 如果类名过多或命名不规范，可能混乱
ID 选择器#header选中 id 为 header 的元素✅ 精准；❌ 优先级高，不易覆盖，复用性差
通用选择器*选中所有元素✅ 可全局重置样式；❌ 性能差，慎用
属性选择器[type="text"]选中指定属性或属性值的元素✅ 精准，方便表单或状态选择；❌ 复杂选择器可能影响性能

2. 组合选择器 ​

选择器示例用法说明优缺点
后代选择器div p选中 <div> 内的所有 <p>✅ 灵活；❌ 层级深时性能略差
子选择器div > p选中 <div> 的直接子 <p>✅ 精准；❌ 层级改变可能失效
相邻兄弟选择器h1 + p选中紧跟在 <h1> 后的 <p>✅ 精准；❌ 只选一个紧邻元素
普通兄弟选择器h1 ~ p选中同级的所有 <h1> 之后的 <p>✅ 灵活；❌ 层级限制

3. 伪类选择器 ​

选择器示例用法说明优缺点
结构伪类:first-child / :last-child / :nth-child(2n)根据元素位置选中✅ 强大，CSS 布局常用；❌ 结构变化可能失效
状态伪类:hover / :focus / :active根据交互状态选中✅ 交互样式必备；❌ 只针对用户操作状态
UI 伪类:checked / :disabled表单状态控制样式✅ 可替代 JS 控制样式；❌ 浏览器兼容性需注意

4. 伪元素选择器 ​

选择器示例用法说明优缺点
::before / ::afterp::after在元素前/后插入内容✅ 可添加装饰效果，无需额外 DOM；❌ 内容不可交互

5. 分组选择器 ​

选择器示例用法说明优缺点
逗号分组h1, h2, h3同时选中多个元素✅ 避免重复书写；❌ 无层级关系

CSS 优先级（权重） ​

CSS 优先级计算规则通常称为 Specificity，记作 (a,b,c,d)：

选择器类型权重
内联样式 style=""a = 1
ID 选择器b = 1
类、属性、伪类c = 1
标签、伪元素d = 1
!important优先覆盖所有（权重外加特殊处理）
css
p {
  color: red;
} /* (0,0,0,1) */
.class1 {
  color: blue;
} /* (0,0,1,0) */
#id1 {
  color: green;
} /* (0,1,0,0) */

优先级：#id1 > .class1 > p
